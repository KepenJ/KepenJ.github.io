关于近期面试有感

Mar 28, 2016 | 杂感 | Hits

前言

po🐷进入我爱购物网已经快2年了，迭代了快两年的主客户端55BBS和只有一版的旅行小贴士，伴随着经历了 team 的组建、激情且热火的 team work 以及离去，途中能认识了很多非常优秀的小伙伴，能跟他们一起共事我觉得是我的荣幸。在这个月迭代完最后的 v5.2.0 版本之后，po🐷觉得是时候该离开了。

整个跑工作的感想

po🐷整个面试过程大概跑了有半个月，面了有 7~8 家公司，其中不乏朋友推荐和自己较为看好的公司，也斩获了几个 offer。回过头来针对这次跑工作大概有以下一些感想：

  1. 拒绝海投！ 如果你不是奔着任意一份工作只要给钱就好的想法的话，即便面试机会寥寥，也不要改变最初的选择和想法，同样的投太多家也会让你应接不暇。
  2. 诚实和不做假！因为po🐷之前面试了很多刚培训出来的人，同样的这一条适用在找工作的人身上。po🐷这里不是看不起刚培训出来的人，毕竟其中不乏优秀的人才，毕竟po🐷的 iOS 最早也是培训出来的。但是造假的简历和不符的工作经验只会让面试官更快的 pass 你，同时也搅乱了整个市场，给之后同样找工作的人带来麻烦。
  3. 专业技能的熟识！这里不光包含自己做过的东西，同样的很多没有用到过项目中的知识点也应该有所涉猎，毕竟工程师是要不断学习和进取才能维持自身价值的。
  4. 面试后的不断总结和汲取！“没有人能一次成功，如果不行那就两次”，一次面试就好比是一次pk过程。你无法保证你能知道或者遇到过面试官所提到的问题，每一次面试你都应该总结一下自己卡到的问题和回忆整个过程中自己的状态，为你下一次进行和好的铺垫和预热。
  5. 乐观的心态！面试这个东西个人认为运气也占一部分，毕竟感情里面还有一见钟情这个说法呢…（逃。随时随地接受被 pass 的结果，这个时候你就需要适当的调整自己和总结问题，只要你清楚自身的实力找到心目中的那一块只是时间的问题。

上面这些总结算是po🐷的一些感受，其中不包含技术方面，如果一定要提到相关技术方面的东西的话有这几块点可以参照一下：

  1. Runtime 这个已经是老生畅谈了，做过 iOS 都应该知道这个，其中不乏一些常见用法和特殊机制，不一定要用到项目当中，但一定要掌握和熟识。
  2. 多线程编程 GCD 、Operation 和 NSThread 之流随着项目做得多，或者优化的深入项目中绝对会用到，熟识也是务必的。
  3. Runloop 相信我这个有必要专门的研究一下，毕竟跟很多东西都有挂钩。优化或者某些需求会用到。
  4. 对 iOS 系统控件或者方法的熟识 说实在的如果你没有事的话，就应该多看看 Apple 给开发提供的文档和代码。
  5. 算法和数据结构 如果你要进相对大和专业一点的 team ，这个是必不可少的，毕竟这个算是程序的灵魂，所有工程师必备技能。但然你不必要精通，但是你也要能跟面试官聊得来。

上述是po🐷觉得有必要的知识点大块，当然整个面试过程中还包含了针对你简历中各种各样的技术点的提问，这个本人觉得应该是不会出问题的。毕竟项目是你自己做的，只要不吹牛逼都不太能出问题。当然作为工程师最重要的还是：谦虚和不断进取的学习态度。

（ps：如果你是个比较内向表达不是很好的工程师的话，那就 Show me your code ！通过 Github StackOverflow 找寻工作是个非常好的选择。）

结尾

总结如下几点：

  1. 明确的目标。
  2. 对自己专业技能的明确和熟识。
  3. 不变的决心和坚持不懈的努力。

po🐷最后还是收到了理想中公司和 team 的 offer。祝愿大家也收获自己心目中的那一个~
