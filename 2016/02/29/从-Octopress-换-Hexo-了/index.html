从 Octopress 换 Hexo 了

Feb 29, 2016 | Hexo | Hits

前言

po🐷之前这个 Blog 是通过 Octopress + Github 搭建的。这次心情一美丽（其实是看到很多朋友的 Blog 都已经换 Hexo 来建站，同时发现有很漂亮的主题…当然 Octopress 长时间不更新也是原因之一）就尝试更换 Hexo 来搭站了。

前期准备工作

如果你是已经大家好了一个个人站，就像po🐷之前已经用 Octopress + Github 建好了一个个人站，那么你其实要做的只是替换 Github 中之前的 XXX.github.io 项目中的内即可。接下来po就从零开始说明一下使用 Hexo + Github 我们如何建站（po🐷用的是 OSX 环境，Windows下可以看下这片博文 《hexo系列教程：（二）搭建hexo博客》）。我们之后也就围绕着下述三个依赖进行讲解

  • Node.js
  • Github
  • Hexo

配置域名（如果你购买了的话）

这里po🐷在万网上面购买了 kepenj.me 这个域名。但是！但是！但是！如果有下一次，我绝对不会再买他们的服务了，改了验证号码接收的手机号码，我足足打了7 8 个电话，到头来也没能解决我的问题。有网友推荐 GoDaddy 准备下次就换他家的，听说口碑不错。

撤了那么多，回归我们的正题。如果你购买了自己的域名，不想在浏览器中输入 xxx.github.io 这么 low 的域名，那么接下来我们就需要设置一下域名管理的解析后台和配置我们的 hexo。

首先我们需要创建一个名为 CNAME 的文本文件，但是注意一点 这个 CNAME 文件要创建到 source 文件夹中，不然当我们上传文件到 Github 时候你会神奇的发现这个文件没有传上去。如下图

文件内容只需要输入你的域名即可，拿po🐷为例，文件中只包含 kepenj.me 这个字符串。

接下来就是在我们所购买的域名后台去设置相关的解析，下图是po🐷在万网后台设置的

其中那些 IP 地址都是指向 Github 的，而那个 kepenj.github.io 则是 Github 原本分配给我们的域名，而我们也正是要用 kepenj.me 来替代这个解析。配置完后大概等个几分钟我们就能正常访问了。

一些 Hexo 使用感受

  • Hexo 正处活跃中，而 Octopress 已经有快1年没人维护了。

  • 及文件目录结构而言，hexo 明显的简明扼要，同样的命令方面也是。

  • 更多且更漂亮的主题，模块化更为明显。改动起来也较为方便。
