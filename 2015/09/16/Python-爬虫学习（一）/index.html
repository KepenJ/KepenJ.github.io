列表（list）、元组（tuple）和字典（dic）

这两个 python 中的数据结构，我第一反应就是跟 Objective－C中的 NSMultableArray 和 NSArray 太像了，你们感受一下

如果说 list 是我们的 MultableArray 那么 tuple 就是我们的 NSArray 了

当然 tuple 是不可变的集合，创建以后就不可更改。但是有一点除外

但是我们改变也只限于影响 tuple 当中 list 和 dic 数据内容，整体的 tuple 数据内容的结构还是没变。其实原因在于 tuple 数据中只是倒入了 list 和 dic 指针，我们无法修改指针，但可以修改指针地址后面指向的数据内容。

既然会存在集合，那字典也势必会有。的确 python 中的 dic 也类似于 OC 当中的 NSMultableDictionary

python 中的函数定义和 C 中略有区别

def + 函数名 ＋参数

举个例子

相似在于它的定义和使用方式上，不同则在于 def 这个应该是 python 给我们封装好的函数创建方法，我们无需再定义函数时，先确认返回参数的类型。

但是 python 在“参数”这块是做了些文章

  • 必备参数：调用时传入参数的个数和顺序和定义时一致

  • 命名参数：个数和顺序和定义时可以不一致。

  • 缺省参数：调用时没有传入的参数，就会使用定义好的缺省值

  • 不定长参数：使用 * 来存储输入的任何参数
