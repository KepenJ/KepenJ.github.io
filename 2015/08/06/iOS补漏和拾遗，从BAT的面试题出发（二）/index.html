设计一个图片缓存机制。怎么存储数据，用什么当key值?

提到图片的缓存，不得不说到 SDWebImage图片请求和处理框架。其中 UIImageView+WebCache 头文件里面的这个方法就是我门常用的 - (void)sd_setImageWithURL:(NSURL *)url 方法的实现，大体分为以下几个步骤：

  1. 如果有 placeholder 的设置，会先加载 placeholder 也就是我们的预加载图片。
  2. 判断是否存在 URL 对象，不存在则返回 error 对象信息。
  3. 当URL存在的下，通过-(id)downloadImageWithURL:options: progress:completed:，返回一个 NSOperation 对象，并将其加入 NSOperationQueue。
  4. NSOperation 对象执行完成之后回调返回image就是我们所需要的图片，完后进行赋值和显示。

其中在 -(id<SDWebImageOperation>)downloadImageWithURL:options: progress:completed: 这个方法中就是我们核心的针对图片的下载和缓存功能：

  1. 判断 URL 的一个可用性。
  2. 用 URL 作为 Key 值，从本地缓存读取 image 对象。
  3. 有 image 对象就返回，没有则调用 manager 的- (id )downloadImageWithURL:options:progress:completed:进行图片请求操作。
  4. 请求成功对图片进行缓存操作。

关于iOS中的cookie?

cookie 在是为了解决http无法记录网络状态而提出的一个机制。这个机制好比是一个“身份证”，原本http是一个无状态的协议，一旦数据交换完毕，客户端与服务器端的连接就会关闭，再次交换数据需要建立新的连接。这就意味着服务器无法从连接上跟踪会话。

消息发送大概有以下几个过程：

  1. 当消息发送给一个对象时，objc_msgSend通过对象的isa指针获取到类的结构体，通过传进来的SEL查询接收者self对象的objc_cache，查询是否缓存过这个方法的调用。有就直接调用相应的IMP进行处理并返回结果id。
  2. 如果对象objc_cache中未有缓存内容，则查询其方法列表结构体指针objc_method_list中是否有对应的方法，有则进行处理并返回结果并缓存所调用的方法。
  3. 如果self自身对象中未发现SEL内容，则查询该类的父类方法。
  4. 再次执行1～3步骤，直至NSObject类。

什么时候处理多线程，几种方式，优缺点。

项目中最常用到多线程的地方：网络请求、数据的存储和读取、多图绘制…因为UI的绘制是在主线城中进行，为了用户体验（也就是所谓的不卡UI），项目中会使用到多线程目的就是为了将一些会消耗大量时间的操作，从main线程中剥离出的一种方案。

Apple 给开发者提供了三种：

  • Thread
  • Operation Queues
  • CGD

自上而下的抽象程度越来越高，同样的使用起来则越来越方便。

NSThread理解起来比较方便，但是用起来则不一样。需要自己去创建和管理线程，遇到多个线程处理逻辑的时候需要自行手动加Lock来防止线程之前的冲突。项目中往往较少会用到NSThread，大多只是些简单创建单一线程，而这样的需求往往NSObject类提供的performSelector:方法或其他类似方法就可以实现。

NSOperation 和 NSOperationQueue 的结合共同实现了我们的 Operation Queues ，相对NSThread需要自己生成和管理线程逻辑，同时还要注意消耗和冲突等问题。NSOperation和NSOperationQueue只需要开发者专注于NSOperation里面的逻辑实现，再将实现好的子类NSOperation加入到NSOperationQueue中设置优先级、同步或者异步、串行或者并发等等，线程的创建和销毁则交由系统自行去处理。更多关于NSOperation和NSOperationQueue的详细介绍可以参照iOS 并发编程之 Operation Queues。

GCD 的底层API是直接由C实现的，而 Thread 合 Operation Queues 底层则都为OC对象。所以相对而言GCD的执行效率相对会高，同时在iOS4.0之后， Operation Queues 的底层实现都是由GCD来完成的。那么如何在GCD和 Operation Queues 或者 Thread 之间做出选择呢，有以下几条建议作为参考：

  1. 如果你想高度监听或者操纵某个或者多个线程的情况的话 Operation Queues 相对GCD而言优势会大很多。
  2. 如果你想让你的代码简洁和高效的话CGD会是一个不错的选择。
  3. NSThread的使用，往往会伴随着Runloop或者Timer。NSThread和performSelector:我会选择后者用到项目里面，当然仅限于最简单的多线程情况。
