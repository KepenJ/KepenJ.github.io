使用 CocoaPods 进行第三方库管理

Mar 19, 2015 | iOS | Hits

关于 CocoaPods

在 CocoaPods 出现之前，广大开发者还在使用手动引入、处理依赖的管理方式来使用第三方框架。就拿我们常用的第三方网络框架 AFNetworking 为例。

手动引入 AFNetworking 原文件的同时还需要在项目当中 import 如下的系统框架，来解决 AFNetworking 的依赖问题。

并且每当 AFNetworking 框架有新的变动或者改变的时候，开发者都需要实时手动变更这些依赖元素。这也只是 AFNetworking 这一个框架，往往一个成型的项目或多或少都会包含 2～3 乃至更多的第三方框架（如果你的开发团队足够支撑所有框架的编写，不排除不需要使用第三方的）。

同时其他的第三方框架所设计的依赖或者 Xcode 配置调整往往有的会非常麻烦。这个时候！一道闪电劈下，CocoaPods 闪亮登场！它的出现，就解决了这些问题。通过配置添加 CocoaPods ，开发者可以非常方便的管理、更新自己项目当中所用到的大部分知名的第三方框架。

安装 CocoaPods

关于 CocoaPods 的安装方法，特别是在天朝这个特殊的环境下依然有很多的 Blog 或者帖子来介绍 CocoaPods 的安装。在这里我就简短说明步骤：

  1. 修改 CocoaPods 基于 gem 安装的源 URL 地址。
  2. 再通过 sudo gem install cocoapods 安装 CocoaPods 。

CocoaPodTest.xcworkspace 是 CocoaPods 给我们创建的 Xcode Workspace 文件，简单通俗来讲 Workspace 的作用就是，当我们的项目需要使用到另一个项目或者模块（这里的项目都是 project 级的），而无需生成静态或者动态库。

Podfile 这个文件就是我们之前编写配置文件。

Podfile.lock 这个文件是 CocoaPods 给我们生成的用于锁定当前第三方库版本，目的就是为了防止多人同时开发的时候，因为第三方库版本更新的问题导致冲突或者其他问题，更多关于这个文件的可以参考 CocoaPods 官方说明。
