什么情况使用 weak 关键字，相比 assign 有什么不同？

这两个字段最大的区别在于，weak 标记了的属性对象被赋的值释放后会直接制空，而 assign 则不会。
assign 常被用于标记 float ， NSIntger 之类的属性。 MRC 下被 assgin 标记的对象实际等于 unsafe_unretained 。在我之前整理的 《记assign引发的血案》中有记录这两个属性的差异。

runtime 如何实现 weak 属性?

被weak标记的属性，底层需要实现两个过程：赋值和释放。

首先是赋值过程：

  1. 创建一个objc对象
  2. objc对象初始化赋值，也就是objc=0
  3. 调用objc_storeWeak(&objc,tempObjc)进行存储。其中tempObjc参数赋值对象的地址作为键值，将附有weak标识的objc变量地址注册到weak表中。如果tempObjc为0，则为释放操作。

下来就是释放过程：

  1. 调用objc_release释放weak对象
  2. 因为引用计数为0所以执行dealloc
  3. _objc_rootDealloc
  4. object_dispose
  5. objc_destructInstance
  6. objc_clear_deallocating

这个问题其实在《Objective-C高级编程:iOS与OS X多线程和内存管理》中针对 __weak 作了非常详细的解释，或者可以查看这篇 《__weak如何实现对象值自动设置为nil的》 的Blog。

+load 和 +initialize 两个方法的用法

+load 方法是当类或分类被添加到 Objective-C runtime 时被调用的，实现这个方法可以让我们在类加载的时候执行一些类相关的行为。

+initialize 方法是在类或它的子类收到第一条消息之前被调用的，这里所指的消息包括实例方法和类方法的调用。

这两个都是用于类的初始化。

loadView 方法的作用。

用于创建ViewController初始化view所调用的方法，xcode4以后就隐藏了。可在这个方法里面顺带创建view上面的各种自控件。之后在vc的view为nil的时候就会调用。一般在MemoryWarning的情况下会置空不在当前屏幕上显示的view。

vc.view = nil; //可以选择置空view或者view上面的自控件，来降低内存的占用。

怎么调试 EXC_BAD_ACCESS 错误.

设置里面勾选上 ZombieO bjects。当设置ZombieO bjects环境变量后，一个对象销毁时会被转化为_NSZombie，当你向一个已经释放的对象发送消息，这个对象就不会向之前那样Crash或者产生一个难以理解的行为，而是放出一个错误消息，然后以一种可预测的可以产生debug断点的方式消失， 因此我们就可以找到具体或者大概是哪个对象被错误的释放了。

SEL 和 IMP 的区别.

SEL 就是我们统称的方法选择器，在 runtime 里面是这样的：

typedef struct objc_selector *SEL;

说白了就是映射到方法的一个字符串。不同类中相同名字的方法所对应的方法选择器是相同的，即使方法名字相同而变量类型不同也会导致它们具有相同的方法选择器，这是需要注意的一点。

IMP 个人理解就是函数指针，他在 runtime 里面则是：

typedef id (*IMP)(id, SEL, ...);

是方法的最终执行地方，编译器在编译的时候就会生成里面的实现代码。他的传入参数当中就包含了SEL和其他定义的参数。所以说在一个类中，通过SEL和其方法的传入参数，是可以确定唯一的一个IMP的。
